# Задание
Разработать сервис автодополнения (autocomplete) адресов на основе данных ФИАС (или КЛАДР).
### Функциональные требования
2. Загрузить ФИАС в PostgreSQL любым способом.
2. Реализовать autocomplete: при вводе текста отправляются запросы на сервер, сервер возвращает подходящие адреса из БД.
3. Серверная часть на PHP принимает запросы, обращается к PostgreSQL, возвращает JSON.
4. При нажатии «Выбрать» выбранный адрес отображается под полем ввода.

# Функциональные возможности

## 1. Загрузка адресных данных

### ФИАС (Федеральная информационная адресная система)

- Автоматическое получение ссылки на актуальный архив через API `https://fias.nalog.ru/WebServices/Public/GetAllDownloadFileInfo`.
- Поддержка двух форматов: DBF (предпочтительный) и XML (потоковый парсинг для больших архивов).
- Фильтрация по региону (код региона, например `59` для Пермского края).
- Пакетная вставка в БД (1000 записей за раз).
- Автоматическая очистка временных файлов после загрузки.

**Команда:**  
`php yii fias/load --region=59`

### КЛАДР (Классификатор адресов России)

- Загрузка архива `.7z` из официального источника или из локального файла.
- Распаковка 7‑zip архивом.
- Чтение DBF-файлов: `KLADR.DBF` (словарь), `STREET.DBF` (улицы), `DOMA.DBF` (дома).
- Восстановление полного адреса по иерархии кодов.
- Фильтрация по региону (двухзначный код, `00` - все регионы).
- Пакетная вставка.

**Команда:**  
`php yii kladr/load 59` - загрузить Пермский край из официального источника  
`php yii kladr/load 00 --local=/path/to/kladr.7z` - загрузить все регионы из локального архива

> **Примечание для релиза 1.0.0:** дома загружаются «как есть» (составные номера не разбиваются).

## 2. API автодополнения (autocomplete)

Серверная часть реализована на Yii2, возвращает JSON.

### Быстрый префиксный поиск

- **Эндпоинт:** `GET /address/autocomplete?q={query}&limit={limit}`
- **Алгоритм:** `lower(full_address) LIKE 'нормализованный_префикс%'`
- **Индекс:** `idx_address_lower_prefix` (текстовый паттерн)
- **Кэш:** Redis, TTL 1 час
- **Ограничения:** минимум 2 символа, максимум 50 результатов.

### Медленный поиск по сходству (для опечаток)

- **Эндпоинт:** `GET /address/autocomplete-slow?q={query}&limit={limit}`
- **Алгоритм:** оператор `%` (pg_trgm) + сортировка по `similarity()`
- **Индекс:** GIN (`idx_address_full_address_gin`)
- **Кэш:** Redis, TTL 6 часов
- **Ограничения:** минимум 3 символа.

### Получение адреса по ID

- **Эндпоинт:** `GET /address/{id}`
- **Возвращает:** полную запись из БД (все поля)
- **Кэш:** Redis, TTL 1 час.

### Статистика

- **Эндпоинт:** `GET /address/stats`
- **Возвращает:** общее количество адресов в базе и дату последнего обновления.
- **Кэш:** Redis, TTL 24 часа.

## 3. Веб-интерфейс

- Простая HTML-страница с полем ввода (`/`).
- Автодополнение при вводе (дебаунс 300 мс).
- Сначала отображаются быстрые результаты, затем подгружаются похожие (slow) асинхронно.
- Возможность выбрать адрес - выбранный адрес отображается под полем.
- Отображение статистики базы адресов.

## 4. Технические особенности

- **Язык:** PHP 8.1
- **Фреймворк:** Yii 2
- **База данных:** PostgreSQL 14 с расширением `pg_trgm`
- **Кэш:** Redis 7
- **Контейнеризация:** Docker Compose
- **Сборка:** Makefile для типовых операций
- **Тестирование:** Codeception (юнит, функциональные, acceptance)

## 5. Инструменты разработчика

- **Загрузка данных:** `make load-fias REGION=59`, `make load-kladr REGION=59`
- **Миграции:** `make migrate`
- **Запуск тестов:** `make test`
- **Вход в контейнер:** `make bash`

# Установка и запуск

## Предварительные требования

- Docker Engine (≥ 24.0) и Docker Compose (≥ 2.20)
- Git
- Make (опционально, но рекомендуется)

## Быстрый старт

1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd <project-directory>
   ```
2. Скопируйте файл окружения и настройте при необходимости:
   ```
   cp .env.example .env
   ```
3. Запустите контейнеры:
   ```
   make up
   ```
   или вручную
   ```
   docker compose up -d
   ```
4. Выполните миграции БД:
   ```
   make migrate
   ```
5. Загрузите адресные данные (пример для Пермского края через ФИАС):
   ```
   make load-fias REGION=59
   ```
   или кладр
   ```
   make load-kladr REGION=59
   ```
6. Откройте в браузере `http://localhost:8080`

## Загрузка данных
### ФИАС
- Регион указывается двухзначным кодом (например, `59`, `77`, `50`).
- Команда автоматически выберет источник (сначала DBF, затем XML).
- Временные файлы сохраняются в `runtime/fias_temp` и удаляются после завершения.
### КЛАДР
- Можно указать локальный архив: `--local=/absolute/path/to/kladr.7z`
- Для загрузки всех регионов используйте 00 в качестве региона.
- При загрузке из официального источника требуется установленный `aria2` и `p7zip` (входят в Docker-образ).

## Проверка работоспособности
- Откройте `http://localhost:8080` - должна отобразиться страница поиска.
- Введите любой адрес, например `Пермский край, г Пермь, ул Ленина` - появятся подсказки.
- Убедитесь, что статистика внизу страницы показывает количество загруженных адресов.

## Устранение неполадок
- **Ошибка подключения к БД:** проверьте, что контейнер `db` запущен и переменные в `.env` совпадают с `docker-compose.yml`.
- **Нет подсказок:** убедитесь, что данные загружены (`make migrate` и команды загрузки выполнены успешно).
- **Медленная загрузка:** увеличьте ресурсы Docker (CPU/RAM). Для загрузки полного ФИАС требуется терпение.
- **Ошибка `aria2c: command not found`:** она не должна возникать внутри Docker-контейнера - образ содержит `aria2`. Если запускаете команды локально, установите `aria2` и `p7zip-full` вручную.

# API документация

Базовый URL: `http://localhost:8080` (или ваш домен)

Все эндпоинты возвращают JSON.

## 1. Быстрый префиксный поиск

Используется для мгновенных подсказок при вводе текста.

**Endpoint:**  
`GET /address/autocomplete`

**Параметры:**

| Параметр | Тип    | Обязательный | По умолчанию | Описание                          |
|----------|--------|--------------|--------------|-----------------------------------|
| `q`      | string | да           | –            | Поисковый запрос (минимум 2 симв.)|
| `limit`  | int    | нет          | 10           | Максимум результатов (1–50)       |

**Пример запроса:**  
`GET /address/autocomplete?q=пермский+край&limit=5`

**Пример ответа:**
```json
[
  {
    "id": 12345,
    "full_address": "Пермский край, г Пермь, ул Ленина"
  },
  {
    "id": 12346,
    "full_address": "Пермский край, г Пермь, ул Кирова"
  }
]
```
**Код ошибки:** `200 OK` (при невалидном запросе возвращается `[]`).

## 2. Медленный поиск по сходству
Используется для исправления опечаток и неполных вводов. Вызывается асинхронно после быстрого поиска.
**Endpoint:**
`GET /address/autocomplete-slow`
**Параметры:**

| Параметр  | Тип     | Обязательный  | По умолчанию  | Описание                            |
|-----------|---------|---------------|---------------|-------------------------------------|
| q         | 	string | 	да           | 	–            | 	Поисковый запрос (минимум 3 симв.) |
| limit     | 	int    | 	нет          | 	10           | 	Максимум результатов (1–50)        |

**Пример запроса:**
`GET /address/autocomplete-slow?q=пермскии+краи&limit=5`
**Пример ответа:** (аналогичен быстрому поиску)
